# Changelog

所有對「gongwen-format」公文格式排版工具的顯著變更記錄於此文件。

格式基於 [Keep a Changelog](https://keepachangelog.com/zh-TW/1.0.0/)，版本號遵循 [語意化版本](https://semver.org/lang/zh-TW/)。

---

## [2.1.0] - 2026-07-23

### 新增
- **智能空段刪除**：附件前保留1行、附件與落款間保留1行、文末保留1行，保留的空段統一設為28pt行距
- **附件懸掛縮進分級**：
  - 1個附件（無序號）→ 左縮進0 + 首行2字符
  - 附件1 → 左縮進6.5字符 + 懸掛4.5字符（數字在2字符處，續行在6.5字符處）
  - 附件2+ → 左縮進6.5字符 + 懸掛1.5字符（數字在5字符處）
- **附件文本規範化**：自動去除書名號《》、末尾標點（。，．,;；）
- **序號點半角轉全角**：三級/四級標題和附件段落中的 `1.` → `1．`，正文不受影響
- **圖片標題識別**：以「架構圖／流程圖／示意圖／效果圖」等結尾的段落 → 居中 + 仿宋小四 + 無縮進
- **無編號一級標題修復**（`_fix_unnumbered_headings`）：
  - 熱啟動：已有`一、`標題時，向後掃描提升後續短正文
  - 冷啟動：文檔無任何`一、`時，自動檢測並提升標題（過濾以「主要」開頭、以「：」結尾的段落）
- **連續附件編號修復**（`_fix_sequential_attachments`）：自動將`附件：1.xxx`後的`2.xxx`、`3.xxx`重新標記為附件
- **佔位日期支持**：`2024年6月XX日`等草稿佔位格式正確識別為落款日期
- **多段標題合併**（`_ensure_title_paragraph`）：首段為空行時，自動將後續多段短正文合併為標題（如「關於開展…」+「項目立項的請示」）
- **WPS兼容縮進單位**（`_set_indent_chars`）：同時設置`w:left`/`w:leftChars`雙單位，避免WPS中Chars單位覆蓋磅值

### 修正
- 一級標題對齊：`JUSTIFY` → `LEFT`（黑體三號左對齊）
- 落款識別：修復 `post_process_signatures` 短路bug（`labels[i] not in ('date_signature',)`）及調用順序（移至所有重分類之後）
- 附件WPS兼容：新增 `_set_indent_chars()` 確保Chars單位不覆蓋磅值
- 空段檢測：改用段落序號而非body元素索引（排除表格干擾）
- 首段空行時標題識別失敗：新增冷啟動模式

### 變更
- `remove_empty_paragraphs`：從「全部刪除」→「智能保留」，並為保留的空段設定28pt行距
- `apply_attachment_format`：參數從 `(paragraph)` → `(paragraph, attachment_index, total_attachments)`
- `HEADING_3_PATTERN`：正則從 `^\d+[\.\、]` → `^\d+[\.\、\．]`（支援全角句點）
- SKILL.md：移除 `present_files` 調用指令，改為僅告知輸出路徑

## [2.0.0] - 2026-07-22

### 新增
- 基於 GB/T 9704 黨政機關公文格式標準的自動排版
- 支援文檔類型識別：標題、主送機關、一至四級標題、正文、附件、落款、版記、簽發人、發文字號
- `--preview` 診斷預覽模式
- `--fix-hierarchy` 層級跳躍修正
- `--title-break=N` 長標題斷行
- `--auto` 自動排版模式
- 引號規範化、項目符號規範化、頁邊距設置
- 表格格式化：仿宋小四、表頭加粗、細單線邊框
