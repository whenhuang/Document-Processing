# 公文格式排版

按《党政机关公文格式》(GB/T 9704) 标准，将未排版的 .docx 文档自动排版为规范公文格式。

## 快速开始

```bash
# 安装依赖
pip install -r requirements.txt

# 预览文档结构
python format_gongwen.py 报告草稿.docx --preview

# 正式排版（输出 报告草稿_已排版.docx）
python format_gongwen.py 报告草稿.docx
```

## 功能特性

- 自动识别文档结构：标题、主送机关、一至四级标题、正文、附件、落款、发文字号、签发人、版记等
- 逐段应用对应的字体、字号、行距、缩进（见下方排版规范表）
- 自动修正页边距、行距（28pt 固定值）、段前段后归零
- 自动添加奇偶页页码（宋体四号，"— 1 —" 格式）
- 自动规范化半角引号为全角配对引号
- 自动拆分标题+正文混合段落
- 自动删除段间空行
- 检测并修正层级跳跃（如"一、"下直接出现"1、"）
- 表格统一格式（仿宋小四、表头加粗、细单线边框）

## CLI 参数

| 参数 | 作用 |
|------|------|
| `--preview` | 仅打印结构预览和诊断报告，不生成排版文件 |
| `--title-break=<N>` | 指定标题断行位置（字符索引），如 `--title-break=18` |
| `--fix-hierarchy` | 自动修正层级跳跃，将跳级的 `1、→（一）` 格式修正 |
| `--auto` | 自动模式：跳过所有交互确认，使用默认行为（适合批处理） |

## 使用示例

```bash
# 预览文档结构，检查层级识别是否正确
python format_gongwen.py 报告草稿.docx --preview

# 排版 + 层级编号修正
python format_gongwen.py 报告草稿.docx --fix-hierarchy

# 指定标题断行位置 + 层级修正
python format_gongwen.py 报告草稿.docx --title-break=18 --fix-hierarchy

# 自动模式（适合脚本/CI 批处理）
python format_gongwen.py 报告草稿.docx --fix-hierarchy --auto

# 指定输出路径
python format_gongwen.py 报告草稿.docx 报告定稿.docx
```

## 排版规范参考

### 对齐总则

| 元素 | 字体 | 字号 | 对齐 | 缩进 |
|------|------|------|------|------|
| 标题 | 方正小标宋简体 | 二号(22pt) | 居中 | 无 |
| 主送机关 | 仿宋_GB2312 | 三号(16pt) | 左对齐 | 顶格 |
| 正文 | 仿宋_GB2312 | 三号(16pt) | 两端对齐 | 首行缩进2字符 |
| 一级标题 `一、` | 黑体 | 三号(16pt) | 两端对齐 | 首行缩进2字符 |
| 二级标题 `（一）` | 楷体_GB2312 | 三号(16pt) | 两端对齐 | 首行缩进2字符（加粗） |
| 三级标题 `1.` | 仿宋_GB2312 | 三号(16pt) | 两端对齐 | 首行缩进2字符 |
| 四级标题 `（1）` | 仿宋_GB2312 | 三号(16pt) | 两端对齐 | 首行缩进2字符 |
| 附件 | 仿宋_GB2312 | 三号(16pt) | 左对齐 | 有缩进 |
| 落款单位/日期 | 仿宋_GB2312 | 三号(16pt) | 右对齐 | 无 |
| 联系人 | 仿宋_GB2312 | 三号(16pt) | 左对齐 | 首行缩进2字符 |
| 发文字号 | 仿宋_GB2312 | 三号(16pt) | 居中 | 无 |
| 签发人 | 仿宋_GB2312 | 三号(16pt) | 右对齐 | 无 |
| 版记（抄送/印发） | 仿宋_GB2312 | 四号(12pt) | 左对齐 | 无 |
| 表格 | 仿宋_GB2312 | 小四(14pt) | 左对齐 | 无 |

### 页边距

上 3.7cm / 下 3.5cm / 左 2.8cm / 右 2.6cm（A4 纸）

### 其他

- 全文行距：28 磅固定值
- 段前段后：0
- 页码：宋体四号，"— 1 —" 格式，奇数页右下角，偶数页左下角

## 自动识别规则

脚本按以下优先级识别段落类型：

1. **标题**：文档首段，或以"关于...的报告/请示/函"开头的段落
2. **层级标题**：正则匹配 `一、/（一）/1.或1、/（1）` 等模式
3. **主送机关**：文档前部、长度<25字符、以"："结尾
4. **附件**：以"附件"开头的段落
5. **落款日期**：文档末尾的"XXXX年XX月XX日"
6. **落款单位**：日期上方的非标点短文本
7. **联系人**：含"联系人："的段落
8. **发文字号**：含 `〔YYYY〕…号` 的段落
9. **签发人**：含"签发人"的段落（上行文）
10. **版记**：以"抄送/印发"开头的段落
11. **正文**：其余非空段落

## 注意事项

- 仅支持 `.docx`（OOXML）格式；旧版 `.doc` 会被自动检测并提示转换
- 需系统安装对应字体：方正小标宋简体、仿宋_GB2312、楷体_GB2312、黑体、宋体
- 源文档只读，所有修改仅作用于输出文件
- 页码需在 Word/WPS 中保存后刷新才能正确显示
- 标题和落款的识别依赖内容特征，结构特殊的文档可能需要手动微调
